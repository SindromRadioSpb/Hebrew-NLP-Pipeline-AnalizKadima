# 2.2. User Flows — KADIMA

> Версия: 2.0 (2026-03-30) | Добавлены flows для аннотации, KB, генерации, Ulpan

---

## Flow 1: Term Extraction Pipeline (v1.0)

```
[Старт] → [Загрузить корпус] → [Выбрать профиль] → [Запустить pipeline]
  → [Просмотр результатов] → [Сравнить профили] → [Экспортировать]
```

### Ветвления
- Корпус не загрузился → ошибка формата → показать поддерживаемые форматы
- Pipeline crashed → показать лог ошибки → предложить retry
- Нет терминов → показать "0 results" → предложить сменить профиль

### Edge cases
- Пустой файл → 0 tokens, 0 terms
- Очень большой файл (>100MB) → progress bar, async processing
- Неподдерживаемый язык → warning "только иврит"

---

## Flow 2: Validation Run (v1.0)

```
[Старт] → [Загрузить gold corpus] → [Запустить pipeline] → [Сравнить expected vs actual]
  → [Заполнить review sheet] → [Получить PASS/WARN/FAIL] → [Экспортировать отчёт]
```

### Ветвления
- Gold corpus не совпадает с pipeline → показать diff
- Manual review items → показать checklist → заполнить вручную
- FAIL → показать конкретные расхождения → рекомендации

---

## Flow 3: Intelligent Annotation (v1.0) ← NEW

```
[Старт] → [Создать проект в Label Studio] → [Загрузить тексты]
  → [KADIMA ML backend делает pre-annotation] → [Аннотатор проверяет/корректирует]
  → [Экспорт аннотаций] → [Gold corpus / Training data]
```

### Ветвления
- Pre-annotation нет → аннотатор размечает с нуля (Label Studio UI)
- Pre-annotation низкого качества → аннотатор исправляет → feedback для дообучения
- Конфликт аннотаций → показать diff → выбрать
- Экспорт как gold corpus → автоматически загружается в M11
- Экспорт как NER training data → запуск дообучения NeoDictaBERT

### Hebrew Templates (Label Studio)
- **NER**: PERSON, LOCATION, ORG, DATE, QUANTITY, MATERIAL, TERM (RTL)
- **Term Review**: valid / invalid / needs_canonical / uncertain
- **POS**: NOUN, VERB, ADJ, ADV, PREP, DET, CONJ, PRON

---

## Flow 4: Term Definition Generation (v1.x) ← NEW

```
[Старт] → [Выбрать термин из KB] → [Dicta-LM генерирует определение]
  → [Показать определение на иврите] → [Аннотатор подтверждает/редактирует]
  → [Сохранить в KB]
```

### Ветвления
- LLM не загружен → fallback: показать только морфологический разбор
- Определение неточное → аннотатор редактирует → сохранить как gold
- Нет контекста → Dicta-LM генерирует общее определение

---

## Flow 5: Knowledge Base Browse (v1.x) ← NEW

```
[Старт] → [Открыть KB] → [Поиск термина] → [Показать: определение, связи, эмбеддинги]
  → [Просмотр связанных терминов] → [Редактировать / добавить]
```

### Ветвления
- Термин не найден → предложить извлечь из корпуса
- Нет определения → предложить сгенерировать через Dicta-LM
- Связи не найдены → предложить вычислить через NeoDictaBERT embeddings

---

## Flow 6: Ulpan Grammar Assistant (v2.0) ← NEW

```
[Старт] → [Стudent вводит предложение на иврите]
  → [KADIMA M3 разбирает морфологию]
  → [Dicta-LM объясняет грамматику на иврите]
  → [Показать: разбор + объяснение + исправления]
  → [Сгенерировать упражнение по паттерну]
```

### Ветвления
- Ошибка в предложении → Dicta-LM указывает ошибку + объясняет правило
- Сложная конструкция → показать биньян/мишкель + примеры
- Генерация упражнений → выбрать конструкцию (смихут, биньян, предлоги) → Dicta-LM генерирует 5 предложений

---

## Flow 7: Active Learning Cycle (v2.0) ← NEW

```
[Старт] → [KADIMA pipeline делает predictions на unlabeled данных]
  → [Uncertainty sampling выбирает неопределённые примеры]
  → [Аннотатор размечает в Label Studio]
  → [Дообучение модели]
  → [Новый цикл с улучшенной моделью]
```

### Ветвления
- Все predictions уверенные → cycle complete → модель готова
- Нет unlabeled данных → предложить загрузить новый корпус
- Дообучение не улучшает → показать метрики → предложить больше данных
