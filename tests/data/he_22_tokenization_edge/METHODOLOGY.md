# Методика: he_22 — Граничные случаи токенизации

## Процесс: Extended Tokenization

### Дополнительные правила (к he_01)
1. **Двойные пробелы**: не создают пустых токенов.
   - "פלדה  ברזל" → 2 токена, не 3.
2. **Maqaf (־)**: еврейский дефис.
   - אל-חלд = 1 токен (compound word).
   -חוזק-משקל = 1 токен.
3. **Geresh (׳)**: часть аббревиатуры.
   - פרופ׳ = 1 токен.
4. **Gershayim (״)**: часть аббревиатуры.
   - ד״ר = 1 токен.
5. **Tab**: действует как пробел (разделитель).
6. **Смешанные разделители**: תברזל.סגסוגת,אלומיניום — зависит от pipeline.

### Acceptance criterion
- Double spaces → no empty tokens: exact.
- Maqaf compounds: manual_review (1 token или 2?).
- Abbreviations: exact (1 token each).
