# Методика: he_04 — N-gram и NP паттерны

## Процесс 1: N-gram Extraction (извлечение н-грамм)

### Вход
Массив токенов (после tokenization).

### Правила
1. **Bigram**: пара последовательных токенов (token[i], token[i+1]).
2. **Trigram**: тройка последовательных токенов (token[i], token[i+1], token[i+2]).
3. Boundary предложения — граница n-gram: n-gram не пересекает границу предложения.
4. Punctuation-only токены разрывают n-gram.

### Выход
Массив n-грамм с позициями и частотами.

### Acceptance criterion
- Наличие/отсутствие конкретного n-gram: EXACT.
- Частота n-gram: EXACT.

## Процесс 2: NP Chunk Extraction (извлечение именных групп)

### Вход
Массив токенов с POS-тегами.

### Паттерны (допустимые NP)
1. **NOUN + NOUN**: חוזק מתיחה, שכבת הגנה, בדיקת איכות.
   - Первый элемент: существительное в конструктной форме (סמיכות).
   - Второй элемент: существительное в абсолютивной форме.
2. **NOUN + ADJ**: מתכת חזקה, טמפרטורה גבוהה.
   - Согласование: ADJ согласуется с NOUN в роде и числе.
3. **ADJ + NOUN**: (редкий паттерн в иврите, обычно NOUN+ADJ).
4. **NOUN + ADJ + NOUN**: (составные паттерны).
5. **DET + NOUN + ADJ**: הפלדה המוקשחת — NP с определённым артиклем.
6. **NOUN + PP**: בדיקה בטמפרטורה גבוהה — NP с предложной группой.

### Правила DET blocking
- DET (артикль ה) может стоять только в начале NP.
- Если DET стоит не в первой позиции — NP невалиден.
- Пример: *מתכת הגדולה — невалиден (DET не первый).

### Выход
Массив NP chunk'ов с boundaries (start_token, end_token).

### Acceptance criterion
- Наличие/отсутствие конкретного NP pattern: EXACT.
- Boundary detection: manual_review (зависит от POS accuracy).
