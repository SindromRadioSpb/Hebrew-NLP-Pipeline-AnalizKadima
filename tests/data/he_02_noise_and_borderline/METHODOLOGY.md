# Методика: he_02 — Фильтрация шума

## Процесс: Noise Classification

### Вход
Токен (после tokenization).

### Правила классификации
1. **Чистая пунктуация**: токен содержит только символы пунктуации (.,;:!?()[]{}«»-–—). Не является Hebrew word.
2. **Числовой**: токен содержит только цифры и десятичный разделитель (2024, 3.14, 100).
3. **Латиница**: токен содержит только ASCII буквы (GDP, NATO, HTTP). Не Hebrew.
4. **Химическая формула**: токен содержит буквы+цифры латиницей (Fe2O3, H2O, CO2).
5. **Количество+единица**: число немедленно followed by unit без пробела (5kg, 200ml, 3.5mm).
6. **Математический**: содержит математические символы (a+b=c, √2, π, ∑).
7. **Hebrew с пунктуацией**: содержит Hebrew буквы, possibly с пунктуацией — это НЕ шум.

### Выход
Для каждого токена: classification tag (noise_type или non_noise).

### Acceptance criterion
- Hebrew слова НЕ помечаются как noise.
- Pure noise items (числа, латиница) помечаются как noise.
- Пограничные cases (quantity+unit) — manual_review.

---

## Процесс: Noise Filtering

### Вход
Массив токенов с classification tags.

### Правила
1. noise-токены исключаются из дальнейшей обработки (lemma extraction, term extraction).
2. non_noise-токены передаются дальше.
3. Пограничные решения зависят от профиля (precise — строже, recall — мягче).

### Выход
Отфильтрованный массив токенов.
