# Методика: he_01 — Разбиение на предложения, токенизация, лемматизация, DET

## 1. Процесс: Sentence Splitting (разбиение на предложения)

### Вход
Текстовый файл (.txt), содержащий Hebrew текст с пунктуацией.

### Правила
1. Boundary-символ — точка `.` (U+002E), стоящая после символа Hebrew буквы или цифры.
2. Точка, следующая за пробелом, считается boundary предыдущего предложения.
3. Каждый нетривиальный фрагмент между двумя boundary-символами = одно предложение.
4. Пустые фрагменты (только пробелы/пустая строка) не считаются предложениями.

### Выход
Массив предложений. Каждое предложение = подстрока исходного текста.

### Edge cases (не в этом корпусе, см. he_15)
- Аббревиатуры с точкой (פרופ׳, ד״р) — точка НЕ boundary
- Десятичные числа (3.14) — точка НЕ boundary
- Многоточие (...) — зависит от реализации

### Acceptance criterion
Точное количество предложений для каждого файла. EXACT.

## 2. Процесс: Tokenization (токенизация)

### Вход
Строка текста (предложение или весь текст).

### Правила
1. Разделитель токенов — пробел (U+002E space).
2. Каждый непустой фрагмент между пробелами = один токен.
3. Знаки препинания, приклеенные к слову — часть токена.
   - Пример: `פלדה.` = 1 токен, не 2.
   - Пример: `(חומר)` = 1 токен, не 3.
4. Пустые строки или строки только из пробелов = 0 токенов.

### Выход
Массив токенов (строк).

### Acceptance criterion
Точное количество токенов для каждого файла. EXACT.

---

## 3. Процесс: DET Detachment (отсоединение определённого артикля)

### Вход
Токен, начинающийся с символа ה (U+05D4, Hebrew Letter He).

### Правила
1. Если токен начинается с ה И длина токена > 2 символов:
   - Попытка: удалить первый символ ה.
   - Проверка: оставшаяся строка — валидное Hebrew слово (содержит хотя бы одну Hebrew букву).
   - Если да: токен расщепляется на {DET: "ה", base: остаток}.
2. Исключения (НЕ отсоединять):
   - ה является частью корня: היתוך, הרכב, הוסיף, הוא.
   - Определение: если без ה слово не существует в языке или имеет другое значение — НЕ отсоединять.
3. Если токен начинается с ה и длина ≤ 2: НЕ отсоединять (местоимения: הוא, הם).

### Выход
Для каждого DET-токена: пара (surface, base).
Для не-DET-токена: (surface, surface).

### Acceptance criterion
Точное количество DET surfaces и точный список поверхностей. EXACT.

## 4. Процесс: Lemma Extraction (лемматизация)

### Вход
Токен (possibly после DET detachment).

### Правила
1. Существительные:
   - Форма ед.ч., м.р. (если есть) = lemma.
   - מעלות → מעלה, חומרים → חומר, סגסוגות → סגסוגת.
   - Конструктные формы: טמפרטורת → טמפרטורה, בדיקת → בדיקה.
2. Прилагательные:
   - Форма ед.ч., м.р. = lemma.
   - גבוהה → גבוה, חזקות → חזק, מחמירים → מחמיר.
3. Глаголы:
   - Инфинитив в базовой форме = lemma.
   - Конкретная форма зависит от морфологического словаря pipeline.
   - משמשת → שימש (или משש, зависит от словаря).
   - נבדק → נבדק (nif'al = base).
4. Местоимения:
   - Базовая форма: הוא, הם.
   - היא → הוא.
5. Числа:
   - Цифровые: lemma = surface (520 → 520).
   - Словесные: lemma = surface (אלף → אלף).
6. Предлоги/союзы:
   - lemma = surface (של →של, את →את).

### Выход
Массив лемм. Каждая lemma сопровождается POS-тегом.

### Acceptance criterion
Точное количество уникальных лемм. EXACT.
Точная частота каждой леммы. EXACT (при определённой токенизации).
POS — EXACT (где форма однозначна), manual_review (где омонимия).

---

## Связи между процессами

```
Input text
  → Sentence Splitting → sentences[]
    → Tokenization → tokens[]
      → DET Detachment → (surface, base, is_det)
        → Lemma Extraction → (lemma, pos)
```

Каждый процесс работает на выходе предыдущего.
Ошибка на раннем этапе каскадно влияет на все последующие.
